## 📖 Overview

This project demonstrates a **Border Gateway Protocol (BGP) Hijack Attack** in a simulated network environment. The purpose of the project is to understand how malicious Autonomous Systems (AS) can manipulate BGP routing advertisements to redirect internet traffic and analyze possible mitigation techniques.

The project was developed as part of a **CCNA Networking course** to gain practical experience with routing protocols, network security, and Cisco device configuration.

---

## 🎯 Objectives

- Understand the working principles of Border Gateway Protocol (BGP).
- Simulate a BGP Hijacking attack.
- Observe the impact of incorrect route advertisements.
- Analyze routing behavior before and after the attack.
- Demonstrate possible mitigation strategies.
- Gain practical experience with Cisco networking technologies.

---

## 🏗️ Project Architecture

The network topology consists of multiple Autonomous Systems (AS) connected through BGP.

The project demonstrates:

- Normal BGP Route Advertisement
- Malicious Route Injection
- Traffic Redirection
- Route Hijacking
- Network Recovery

## 🔒 Security Considerations

BGP does not inherently verify route authenticity, making it susceptible to route hijacking.

Possible mitigation techniques include:

- Prefix Filtering
- Route Filtering
- RPKI (Resource Public Key Infrastructure)
- Route Origin Validation (ROV)
- AS Path Filtering
- Maximum Prefix Limits
